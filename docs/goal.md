# Goal

Deliver a clean, maintainable, locally runnable responsive home page implementation from the provided Figma desktop, tablet, and mobile designs.

The implementation should demonstrate semantic HTML, organized CSS and TypeScript, accessible interactions, verified rendering at the target breakpoints, and a README documenting technical decisions and local setup.

## Definition Of Done

- The home page matches the Figma desktop, tablet, and mobile designs with strong visual fidelity.
- The layout works correctly at the Figma target sizes and between breakpoints.
- There is no unintended horizontal scrolling, overlap, broken spacing, or clipped text.
- HTML is semantic, using appropriate landmarks such as `header`, `nav`, `main`, `section`, and `footer`.
- Headings, links, buttons, alt text, focus states, and keyboard behavior follow accessibility basics.
- CSS is organized with a consistent naming convention.
- JavaScript or TypeScript is used only where it adds real behavior.
- The project builds and runs locally with clear commands.
- `README.md` explains the stack choice, setup instructions, local run commands, and relevant implementation notes.

## Out Of Scope

- Backend or API work.
- Authentication.
- CMS or data integration.
- Multi-page routing beyond the home page.
- Large UI libraries unless the design clearly requires them.
- Pixel-perfect behavior for unknown screen sizes beyond sensible responsive interpolation.

## Quality Bar

This is an interview assignment, so the implementation should prioritize responsive judgment, clean structure, accessibility awareness, and maintainable code over unnecessary tooling or over-engineering.
