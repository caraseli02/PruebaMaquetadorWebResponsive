# Diseño Fluido y Tipografía Adaptativa usando CSS clamp()

Para resolver la transición responsiva entre los breakpoints de Figma (Móvil `390px`, Tableta pequeña `744px`, Tableta grande `1023px` y Escritorio `1440px`), decidimos adoptar una estrategia de maquetación fluida utilizando la función nativa `clamp()` de CSS en lugar de definir múltiples breakpoints rígidos y repetitivos en media queries. Esta decisión se fundamenta en la necesidad de ofrecer una visualización óptima y sin saltos bruscos en cualquier tamaño de pantalla intermedio, escalando tipografías, paddings y dimensiones de forma proporcional y automática según el ancho del viewport (usando unidades `vw`), lo que simplifica la base de código CSS y elimina solapamientos de contenido en pantallas no contempladas en los diseños estáticos.
