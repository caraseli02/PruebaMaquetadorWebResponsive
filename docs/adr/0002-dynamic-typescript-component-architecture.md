# Uso de Componentes Dinámicos Programáticos en TypeScript (Vanilla TS)

Para la implementación del catálogo de viajes, filtros y diálogos, decidimos construir componentes modulares en archivos `.ts` que instancian y manipulan el DOM de forma programática (usando `document.createElement`, `replaceChildren` y escuchadores nativos) en lugar de utilizar un framework (como React o Vue) o plantillas estáticas de HTML (`<template>`). Esta decisión equilibra las necesidades de interactividad compleja (filtrado en el cliente y sincronización de estado con `TravelFilterState`) con las restricciones típicas de una prueba técnica: cero dependencias pesadas en producción, rendimiento de carga instantáneo (0 KB de runtime framework), tipado estricto en tiempo de compilación y compatibilidad nativa directa con las historias de Storybook.
